方法

每篇文章保留两层：可读页面与源文档。

网页用于公开阅读和导航，Markdown 用于长期保存、版本管理、复盘、重写、引用和导出。

  1. 对话产生主题
  2. 主题整理成源文档
  3. 源文档转成文章页
  4. 首页目录连接所有文章
  5. Git 记录每一次修订

这是一张会生长的地图，不是一套已经封口的教义。

这个档案馆会随着新主题继续演化。目标不是过早冻结理论，而是保存理论变得更锋利的路径。
